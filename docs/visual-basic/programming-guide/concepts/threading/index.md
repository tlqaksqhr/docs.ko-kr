# <a name="threading-visual-basic"></a><span data-ttu-id="79049-102">스레딩(Visual Basic)</span><span class="sxs-lookup"><span data-stu-id="79049-102">Threading (Visual Basic)</span></span>
<span data-ttu-id="79049-103">스레딩을 사용하면 Visual Basic 프로그램에서 동시 처리를 수행할 수 있으므로 한 번에 여러 작업을 진행할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="79049-103">Threading enables your Visual Basic program to perform concurrent processing so that you can do more than one operation at a time.</span></span> <span data-ttu-id="79049-104">예를 들어 스레딩을 사용하여 사용자의 입력을 모니터링하고, 백그라운드 작업을 수행하고, 동시 입력 스트림을 처리할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="79049-104">For example, you can use threading to monitor input from the user, perform background tasks, and handle simultaneous streams of input.</span></span>  
  
 <span data-ttu-id="79049-105">스레드에는 다음과 같은 속성이 있습니다.</span><span class="sxs-lookup"><span data-stu-id="79049-105">Threads have the following properties:</span></span>  
  
-   <span data-ttu-id="79049-106">스레드를 사용하면 프로그램에서 동시 처리를 수행할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="79049-106">Threads enable your program to perform concurrent processing.</span></span>  
  
-   <span data-ttu-id="79049-107">.NET Framework <xref:System.Threading> 네임스페이스를 통해 스레드를 쉽게 사용할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="79049-107">The .NET Framework <xref:System.Threading> namespace makes using threads easier.</span></span>  
  
-   <span data-ttu-id="79049-108">스레드는 응용 프로그램의 리소스를 공유합니다.</span><span class="sxs-lookup"><span data-stu-id="79049-108">Threads share the application's resources.</span></span> <span data-ttu-id="79049-109">자세한 내용은 [스레드 및 스레딩 사용](https://msdn.microsoft.com/library/e1dx6b2h)을 참조하세요.</span><span class="sxs-lookup"><span data-stu-id="79049-109">For more information, see [Using Threads and Threading](https://msdn.microsoft.com/library/e1dx6b2h).</span></span>  
  
 <span data-ttu-id="79049-110">기본적으로 Visual Basic 프로그램에는 스레드가 하나 있습니다.</span><span class="sxs-lookup"><span data-stu-id="79049-110">By default, a Visual Basic program has one thread.</span></span> <span data-ttu-id="79049-111">그러나 보조 스레드를 만들어 기본 스레드와 함께 코드를 실행하는 데 사용할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="79049-111">However, auxiliary threads can be created and used to execute code in parallel with the primary thread.</span></span> <span data-ttu-id="79049-112">이러한 스레드를 일반적으로 *작업자 스레드*라고 합니다.</span><span class="sxs-lookup"><span data-stu-id="79049-112">These threads are often called *worker threads*.</span></span>  
  
 <span data-ttu-id="79049-113">작업자 스레드는 기본 스레드를 묶어두지 않고서도 시간이 오래 걸리거나 시간이 중요한 작업을 수행하는 데 사용할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="79049-113">Worker threads can be used to perform time-consuming or time-critical tasks without tying up the primary thread.</span></span> <span data-ttu-id="79049-114">예를 들어 작업자 스레드는 이전 요청이 완료될 때까지 기다리지 않고서도 서버 응용 프로그램에서 들어오는 요청을 이행하는 데 종종 사용됩니다.</span><span class="sxs-lookup"><span data-stu-id="79049-114">For example, worker threads are often used in server applications to fulfill incoming requests without waiting for the previous request to be completed.</span></span> <span data-ttu-id="79049-115">또한 작업자 스레드는 사용자 인터페이스 요소를 구동하는 주 스레드가 사용자 작업에 계속 응답하도록 데스크톱 응용 프로그램에서 "백그라운드" 작업을 수행하는 데 사용됩니다.</span><span class="sxs-lookup"><span data-stu-id="79049-115">Worker threads are also used to perform "background" tasks in desktop applications so that the main thread--which drives user interface elements--remains responsive to user actions.</span></span>  
  
 <span data-ttu-id="79049-116">스레딩은 처리량 및 응답성 관련 문제를 해결하지만, 교착 상태 및 경합 상태 같은 리소스 공유 문제를 유발할 수도 있습니다.</span><span class="sxs-lookup"><span data-stu-id="79049-116">Threading solves problems with throughput and responsiveness, but it can also introduce resource-sharing issues such as deadlocks and race conditions.</span></span> <span data-ttu-id="79049-117">여러 스레드는 파일 핸들 및 네트워크 연결 같은 다양한 리소스를 필요로 하는 작업에 적합합니다.</span><span class="sxs-lookup"><span data-stu-id="79049-117">Multiple threads are best for tasks that require different resources such as file handles and network connections.</span></span> <span data-ttu-id="79049-118">단일 리소스에 여러 스레드를 할당하면 동기화 문제가 발생할 수 있으며, 다른 스레드를 기다릴 때 스레드를 자주 차단하면 여러 스레드를 사용하는 목적에 어긋납니다.</span><span class="sxs-lookup"><span data-stu-id="79049-118">Assigning multiple threads to a single resource is likely to cause synchronization issues, and having threads frequently blocked when waiting for other threads defeats the purpose of using multiple threads.</span></span>  
  
 <span data-ttu-id="79049-119">일반적인 전략은 작업자 스레드를 사용하여, 다른 스레드에서 사용하는 많은 리소스를 필요로 하지 않는 시간이 오래 걸리는 작업이나 시간이 중요한 작업을 수행하는 것입니다.</span><span class="sxs-lookup"><span data-stu-id="79049-119">A common strategy is to use worker threads to perform time-consuming or time-critical tasks that do not require many of the resources used by other threads.</span></span> <span data-ttu-id="79049-120">당연히 여러 스레드에서 프로그램의 일부 리소스에 액세스해야 합니다.</span><span class="sxs-lookup"><span data-stu-id="79049-120">Naturally, some resources in your program must be accessed by multiple threads.</span></span> <span data-ttu-id="79049-121">이러한 경우에는 <xref:System.Threading> 네임스페이스에서 스레드 동기화에 대한 클래스를 제공합니다.</span><span class="sxs-lookup"><span data-stu-id="79049-121">For these cases, the <xref:System.Threading> namespace provides classes for synchronizing threads.</span></span> <span data-ttu-id="79049-122">이러한 클래스에는 <xref:System.Threading.Mutex>, <xref:System.Threading.Monitor>, <xref:System.Threading.Interlocked>, <xref:System.Threading.AutoResetEvent> 및 <xref:System.Threading.ManualResetEvent>가 있습니다.</span><span class="sxs-lookup"><span data-stu-id="79049-122">These classes include <xref:System.Threading.Mutex>, <xref:System.Threading.Monitor>, <xref:System.Threading.Interlocked>, <xref:System.Threading.AutoResetEvent>, and <xref:System.Threading.ManualResetEvent>.</span></span>  
  
 <span data-ttu-id="79049-123">이러한 클래스의 일부 또는 모두를 사용하여 여러 스레드 작업을 동기화할 수 있지만, 스레딩에 대한 일부 지원이 Visual Basic 언어에서 지원됩니다.</span><span class="sxs-lookup"><span data-stu-id="79049-123">You can use some or all these classes to synchronize the activities of multiple threads, but some support for threading is supported by the Visual Basic language.</span></span> <span data-ttu-id="79049-124">예를 들어 [SyncLock 문](../../../../visual-basic/language-reference/statements/synclock-statement.md)은 <xref:System.Threading.Monitor>를 암시적으로 사용하여 동기화 기능을 제공합니다.</span><span class="sxs-lookup"><span data-stu-id="79049-124">For example, the [SyncLock Statement](../../../../visual-basic/language-reference/statements/synclock-statement.md) provides synchronization features through implicit use of <xref:System.Threading.Monitor>.</span></span>
