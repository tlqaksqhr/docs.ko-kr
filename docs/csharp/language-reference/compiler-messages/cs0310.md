# <a name="compiler-error-cs0310"></a><span data-ttu-id="e5a25-102">컴파일러 오류 CS0310</span><span class="sxs-lookup"><span data-stu-id="e5a25-102">Compiler Error CS0310</span></span>
<span data-ttu-id="e5a25-103">제네릭 형식 또는 메서드 ‘generic’에서 ‘parameter’ 매개 변수로 사용하려면 ‘typename’ 형식이 매개 변수가 없는 public 생성자를 사용하는 비추상 형식이어야 합니다.</span><span class="sxs-lookup"><span data-stu-id="e5a25-103">The type 'typename' must be a non-abstract type with a public parameterless constructor in order to use it as parameter 'parameter' in the generic type or method 'generic'</span></span>  
  
 <span data-ttu-id="e5a25-104">제네릭 형식 또는 메서드는 해당 where 절에서 새 제약 조건을 정의하므로 해당 제네릭 형식 또는 메서드에 대한 형식 인수로 사용하려면 모든 형식에 매개 변수가 없는 public 생성자가 있어야 합니다.</span><span class="sxs-lookup"><span data-stu-id="e5a25-104">The generic type or method defines a new constraint in its where clause, so any type must have a public parameterless constructor in order to be used as a type argument for that generic type or method.</span></span> <span data-ttu-id="e5a25-105">이 오류를 방지하려면 형식에 올바른 생성자가 있는지 확인하거나 제네릭 형식 또는 메서드의 제약 조건 절을 수정합니다.</span><span class="sxs-lookup"><span data-stu-id="e5a25-105">To avoid this error, make sure that the type has the correct constructor, or modify the constraint clause of the generic type or method.</span></span>  
  
## <a name="example"></a><span data-ttu-id="e5a25-106">예</span><span class="sxs-lookup"><span data-stu-id="e5a25-106">Example</span></span>  
 <span data-ttu-id="e5a25-107">다음 샘플에서는 CS0310을 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="e5a25-107">The following sample generates CS0310:</span></span>  
  
```csharp  
// CS0310.cs  
using System;  
  
class G<T> where T : new()  
{  
    T t;  
  
    public G()  
    {  
        t = new T();  
        Console.WriteLine(t);  
    }  
}  
  
class B  
{  
    private B() { }  
    // Try this instead:  
    // public B() { }  
}  
  
class CMain  
{  
    public static void Main()  
    {  
        G<B> g = new G<B>();   // CS0310  
        Console.WriteLine(g.ToString());  
    }  
}  
```
