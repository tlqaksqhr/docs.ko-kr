<span data-ttu-id="8a364-103">‘function’: ‘property’에 재정의 가능한 get 접근자가 없으므로 재정의할 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="8a364-103">'function' : cannot override because 'property' does not have an overridable get accessor</span></span>  
  
 <span data-ttu-id="8a364-104">기본 클래스에 재정의할 정의가 없을 때 속성 접근자에 대한 재정의를 정의하려고 했습니다.</span><span class="sxs-lookup"><span data-stu-id="8a364-104">A try was made to define an override for a property accessor when the base class has no such definition to override.</span></span> <span data-ttu-id="8a364-105">이 오류는 다음과 같은 방법으로 해결할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="8a364-105">You can resolve this error by:</span></span>

-   <span data-ttu-id="8a364-108">파생 클래스의 속성에 [new](../../../csharp/language-reference/keywords/new.md) 키워드를 추가하여 기본 클래스 속성 숨기기</span><span class="sxs-lookup"><span data-stu-id="8a364-108">Hiding the base class property by adding the [new](../../../csharp/language-reference/keywords/new.md) keyword to a property in a derived class.</span></span>

 <span data-ttu-id="8a364-110">자세한 내용은 [속성 사용](../../../csharp/programming-guide/classes-and-structs/using-properties.md)을 참조하세요.</span><span class="sxs-lookup"><span data-stu-id="8a364-110">For more information, see [Using Properties](../../../csharp/programming-guide/classes-and-structs/using-properties.md).</span></span>  
  
## <a name="example"></a><span data-ttu-id="8a364-111">예제</span><span class="sxs-lookup"><span data-stu-id="8a364-111">Example</span></span>  
 <span data-ttu-id="8a364-112">다음 샘플에서는 CS0545를 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="8a364-112">The following sample generates CS0545.</span></span>  
  
```csharp  
// CS0545.cs  
// compile with: /target:library  
// CS0545  
public class a  
{  
   public virtual int i  
   {  
      set {}  
  
      // Uncomment the following line to resolve.  
      // get { return 0; }  
   }  
}  
  
public class b : a  
{  
   public override int i  
   {  
      get { return 0; }  
      set {}   // OK  
   }  
}  
```
