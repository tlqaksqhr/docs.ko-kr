# <a name="compiler-error-cs0433"></a><span data-ttu-id="f90ac-102">컴파일러 오류 CS0433</span><span class="sxs-lookup"><span data-stu-id="f90ac-102">Compiler Error CS0433</span></span>
<span data-ttu-id="f90ac-103">TypeName1 형식이 TypeName2 및 TypeName3에 모두 있습니다.</span><span class="sxs-lookup"><span data-stu-id="f90ac-103">The type TypeName1 exists in both TypeName2 and TypeName3</span></span>  
  
 <span data-ttu-id="f90ac-104">응용 프로그램에서 참조된 두 가지 어셈블리에 동일한 네임스페이스 및 형식이 있어서 모호성이 발생합니다.</span><span class="sxs-lookup"><span data-stu-id="f90ac-104">Two different assemblies referenced in your application contain the same namespace and type, which produces ambiguity.</span></span>  
  
 <span data-ttu-id="f90ac-105">이 오류를 해결하려면 [/reference(C# 컴파일러 옵션)](../../../csharp/language-reference/compiler-options/reference-compiler-option.md) 컴파일러 옵션의 별칭 기능을 사용하거나, 어셈블리 중 하나를 참조하지 마세요.</span><span class="sxs-lookup"><span data-stu-id="f90ac-105">To resolve this error, use the alias feature of the [/reference (C# Compiler Options)](../../../csharp/language-reference/compiler-options/reference-compiler-option.md) compiler option or do not reference one of your assemblies.</span></span>  
  
## <a name="example"></a><span data-ttu-id="f90ac-106">예제</span><span class="sxs-lookup"><span data-stu-id="f90ac-106">Example</span></span>

```csharp  
// CS0433_1.cs

namespace TypeBindConflicts   
{  
   public class AggPubImpAggPubImp {}  
}  
```  
  
## <a name="example"></a><span data-ttu-id="f90ac-108">예제</span><span class="sxs-lookup"><span data-stu-id="f90ac-108">Example</span></span>  
 <span data-ttu-id="f90ac-109">이 코드는 모호한 형식의 두 번째 복사본으로 DLL을 만듭니다.</span><span class="sxs-lookup"><span data-stu-id="f90ac-109">This code creates the DLL with the second copy of the ambiguous type.</span></span>  
  
```csharp  
// CS0433_2.cs  
// compile with: /target:library  
namespace TypeBindConflicts   
{  
   public class AggPubImpAggPubImp {}  
}  
```  
  
## <a name="example"></a><span data-ttu-id="f90ac-110">예제</span><span class="sxs-lookup"><span data-stu-id="f90ac-110">Example</span></span>  
 <span data-ttu-id="f90ac-111">다음 예제에서는 CS0433을 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="f90ac-111">The following example generates CS0433.</span></span>  
  
```csharp  
// CS0433_3.cs  
// compile with: /reference:cs0433_1.dll /reference:cs0433_2.dll  
using TypeBindConflicts;  
public class Test   
{  
   public static void Main()   
   {  
      AggPubImpAggPubImp n6 = new AggPubImpAggPubImp();   // CS0433  
   }  
}  
```  
  
## <a name="example"></a><span data-ttu-id="f90ac-112">예제</span><span class="sxs-lookup"><span data-stu-id="f90ac-112">Example</span></span>  
 <span data-ttu-id="f90ac-113">다음 예제에서는 **/reference** 컴파일러 옵션의 별칭 기능을 사용하여 이 CS0433 오류를 해결하는 방법을 보여 줍니다.</span><span class="sxs-lookup"><span data-stu-id="f90ac-113">The following example shows how you can use the alias feature of the **/reference** compiler option to resolve this CS0433 error.</span></span>

```csharp  
// compile with: /reference:cs0433_1.dll /reference:TypeBindConflicts=cs0433_2.dll

{  
   public static void Main()   
   {  
      AggPubImpAggPubImp n6 = new AggPubImpAggPubImp();  
   }  
}  
```
