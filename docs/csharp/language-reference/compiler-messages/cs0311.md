<span data-ttu-id="a9160-103">'type1' 형식은 제네릭 형식 또는 '\<name>' 메서드에서 'T' 형식 매개 변수로 사용할 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="a9160-103">The type 'type1' cannot be used as type parameter 'T' in the generic type or method '\<name>'.</span></span> <span data-ttu-id="a9160-104">'type1'에서 'type2'로의 암시적 참조 변환이 없습니다.</span><span class="sxs-lookup"><span data-stu-id="a9160-104">There is no implicit reference conversion from 'type1' to 'type2'.</span></span>  
  
 <span data-ttu-id="a9160-105">제약 조건이 제네릭 형식 매개 변수에 적용될 때는 구체적 인수에서 제약 조건 형식으로의 암시적 ID 또는 참조 변환이 있어야 합니다.</span><span class="sxs-lookup"><span data-stu-id="a9160-105">When a constraint is applied to a generic type parameter, an implicit identity or reference conversion must exist from the concrete argument to the type of the constraint.</span></span>

## <a name="example"></a><span data-ttu-id="a9160-110">예제</span><span class="sxs-lookup"><span data-stu-id="a9160-110">Example</span></span>  
  
```csharp  
// cs0311.cs  
class B{}  
class C{}  
class Test<T> where T : C  
{ }  
  
class Program  
{  
    static void Main()  
    {  
        Test<B> test = new Test<B>(); //CS0311  
    }  
}  
```  
  
 <span data-ttu-id="a9160-111">값 형식 인수를 사용하려고 할 때 이 오류가 발생하면 `short`에서 `int`로의 변환과 같은 암시적 숫자 변환은 제네릭 형식 매개 변수를 충족하지 않는다는 점에 유의하세요.</span><span class="sxs-lookup"><span data-stu-id="a9160-111">If this error occurs when trying to use a value-type argument, notice that an implicit numeric conversion, for example from `short` to `int`, does not satisfy a generic type parameter.</span></span>
