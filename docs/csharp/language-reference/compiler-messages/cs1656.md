 <span data-ttu-id="c1963-104">이 오류는 읽기 전용 컨텍스트에서 변수에 할당할 때 발생합니다.</span><span class="sxs-lookup"><span data-stu-id="c1963-104">This error occurs when an assignment to variable occurs in a read-only context.</span></span> <span data-ttu-id="c1963-105">읽기 전용 컨텍스트에는 [foreach](../../../csharp/language-reference/keywords/foreach-in.md) 반복 변수, [using](../../../csharp/language-reference/keywords/using-statement.md) 변수 및 [fixed](../../../csharp/language-reference/keywords/fixed-statement.md) 변수가 포함됩니다.</span><span class="sxs-lookup"><span data-stu-id="c1963-105">Read-only contexts include [foreach](../../../csharp/language-reference/keywords/foreach-in.md) iteration variables, [using](../../../csharp/language-reference/keywords/using-statement.md) variables, and [fixed](../../../csharp/language-reference/keywords/fixed-statement.md) variables.</span></span> <span data-ttu-id="c1963-106">이 오류를 해결하려면 `using` 블록, `foreach` 문 및 `fixed` 문에서 문 변수에 할당하지 않아야 합니다.</span><span class="sxs-lookup"><span data-stu-id="c1963-106">To resolve this error, avoid assignments to a statement variable in `using` blocks, `foreach` statements, and `fixed` statements.</span></span>  
  
## <a name="example"></a><span data-ttu-id="c1963-107">예</span><span class="sxs-lookup"><span data-stu-id="c1963-107">Example</span></span>  
 <span data-ttu-id="c1963-108">다음 예제에서는 `foreach` 루프 내에 있는 컬렉션의 전체 요소를 교체하려고 하기 때문에 CS1656 오류가 발생합니다.</span><span class="sxs-lookup"><span data-stu-id="c1963-108">The following example generates error CS1656 because it tries to replace complete elements of a collection inside a `foreach` loop.</span></span> <span data-ttu-id="c1963-109">오류를 해결하는 한 가지 방법은 `foreach` 루프를 [for](../../../csharp/language-reference/keywords/for.md) 루프로 변경하는 것입니다.</span><span class="sxs-lookup"><span data-stu-id="c1963-109">One way to work around the error is to change the `foreach` loop to a [for](../../../csharp/language-reference/keywords/for.md) loop.</span></span> <span data-ttu-id="c1963-110">여기에 표시되지 않은 또 한 가지 방법은 기존 요소의 멤버를 수정하는 것인데, 이는 클래스에서는 가능하지만 구조체에서는 가능하지 않습니다.</span><span class="sxs-lookup"><span data-stu-id="c1963-110">Another way, not shown here, is to modify the members of the existing element; this is possible with classes, but not with structs.</span></span>  
  
```csharp  
using System;  
using System.Collections;  
using System.Collections.Generic;  
using System.Text;  
  
namespace CS1656_2  
{  
  
    class Book  
    {  
        public string Title;  
        public string Author;  
        public double Price;  
        public Book(string t, string a, double p)  
        {  
            Title=t;  
            Author=a;  
            Price=p;  
  
        }  
    }  
  
    class Program  
    {  
        private List<Book> list;  
        static void Main(string[] args)  
        {  
            Program prog = new Program();  
            prog.list = new List<Book>();  
            prog.list.Add(new Book ("The C# Programming Language",  
                                    "Hejlsberg, Wiltamuth, Golde",  
                                     29.95));  
            prog.list.Add(new Book ("The C++ Programming Language",  
                                    "Stroustrup",  
                                     29.95));  
            prog.list.Add(new Book ("The C Programming Language",  
                                    "Kernighan, Ritchie",  
                                    29.95));  
            foreach(Book b in prog.list)  
            {  
                // Cannot modify an entire element in a foreach loop   
                // even with reference types.  
                // Use a for or while loop instead  
                if (b.Title == "The C Programming Language")  
                    // Cannot assign to 'b' because it is a 'foreach   
                    // iteration variable'  
                    b = new Book("Programming Windows, 5th Ed.", "Petzold", 29.95); //CS1656  
            }  
  
            //With a for loop you can modify elements  
            //for(int x = 0; x < prog.list.Count; x++)  
            //{  
            //    if(prog.list[x].Title== "The C Programming Language")  
            //        prog.list[x] = new Book("Programming Windows, 5th Ed.", "Petzold", 29.95);  
            //}  
            //foreach(Book b in prog.list)  
            //    Console.WriteLine(b.Title);  
  
        }  
    }  
}  
```  
  
## <a name="example"></a><span data-ttu-id="c1963-111">예</span><span class="sxs-lookup"><span data-stu-id="c1963-111">Example</span></span>  
 <span data-ttu-id="c1963-112">다음 샘플은 `foreach` 루프 이외의 다른 컨텍스트에서 CS1656 오류가 어떻게 발생하는지 보여 줍니다.</span><span class="sxs-lookup"><span data-stu-id="c1963-112">The following sample demonstrates how CS1656 can be generated in other contexts besides a `foreach` loop:</span></span>  
  
```csharp  
// CS1656.cs  
// compile with: /unsafe  
using System;  
  
class C : IDisposable  
{  
    public void Dispose() { }  
}  
  
class CMain  
{  
    unsafe public static void Main()  
    {  
        using (C c = new C())  
        {  
            // Cannot assign to 'c' because it is a 'using variable'  
            c = new C(); // CS1656  
        }  
  
        int[] ary = new int[] { 1, 2, 3, 4 };  
        fixed (int* p = ary)  
        {  
            // Cannot assign to 'p' because it is a 'fixed variable'  
            p = null; // CS1656  
        }  
    }  
}  
```
