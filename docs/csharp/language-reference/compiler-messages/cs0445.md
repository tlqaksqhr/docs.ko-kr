# <a name="compiler-error-cs0445"></a><span data-ttu-id="96fb2-102">컴파일러 오류 CS0445</span><span class="sxs-lookup"><span data-stu-id="96fb2-102">Compiler Error CS0445</span></span>
<span data-ttu-id="96fb2-103">unboxing 변환 결과는 수정할 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="96fb2-103">Cannot modify the result of an unboxing conversion</span></span>  
  
 <span data-ttu-id="96fb2-104">unboxing 변환 결과는 임시 변수입니다.</span><span class="sxs-lookup"><span data-stu-id="96fb2-104">The result of an unboxing conversion is a temporary variable.</span></span> <span data-ttu-id="96fb2-105">임시 변수가 사라지면 수정 내용도 취소되므로 컴파일러는 이러한 변수를 수정할 수 없도록 합니다.</span><span class="sxs-lookup"><span data-stu-id="96fb2-105">The compiler prevents you from modifying such variables because any modification would go away when the temporary variable goes away.</span></span> <span data-ttu-id="96fb2-106">이를 수정하려면 새 값 형식 변수를 선언하여 중간 식을 저장하고 unboxing 변환의 결과를 해당 변수에 할당합니다.</span><span class="sxs-lookup"><span data-stu-id="96fb2-106">To fix this, declare a new value-type variable to store the intermediate expression, and assign the result of the unboxing conversion to that variable.</span></span>  
  
 <span data-ttu-id="96fb2-107">다음 코드에서는 CS0455를 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="96fb2-107">The following code generates CS0455.</span></span>  
  
```csharp  
// CS0445.CS  
class UnboxingTest  
{  
    public static void Main()  
    {  
        Point p;  
        p.x = 1;  
        p.y = 2;  
        object obj = p;  
        // The following line generates CS0445, because the result  
        // of unboxing obj is a temporary variable.  
        ((Point)obj).x = 2;  
  
        // The following lines resolve the error.  
  
        // Store the result of the unboxing conversion in p2.  
        Point p2;       
        p2 = (Point)obj;  
        // Then you can modify the unboxed value.  
        p2.x = 2;  
    }  
}  
  
struct Point  
{  
    public int x, y;  
}  
```
