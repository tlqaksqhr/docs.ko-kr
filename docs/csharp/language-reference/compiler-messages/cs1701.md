# <a name="compiler-warning-level-2-cs1701"></a>컴파일러 경고(수준 2) CS1701
"Assembly Name #1" 어셈블리 참조가 "Assembly Name #2"와 일치하는 것으로 간주합니다. 런타임 정책을 지정해야 합니다.  
  
 두 어셈블리의 릴리스 및/또는 버전 번호가 다릅니다. 통합하려면 응용 프로그램의 .config 파일에서 지시문을 지정하고 다음 예제 코드에서처럼 어셈블리의 강력한 이름을 올바르게 제공해야 합니다.  
  
## <a name="example"></a>예제  
 다음과 같은 다중 파일 샘플은 두 개의 서로 다른 외부 별칭을 사용하여 어셈블리를 참조합니다. 이 첫 번째 샘플에서는 CS1701_d 어셈블리를 만드는 코드의 이전 버전을 빌드합니다.  
  
```csharp  
// CS1701_a.cs  
// compile with: /target:library /out:cs1701_d.dll /keyfile:mykey.snk  
using System.Reflection;  
[assembly:AssemblyVersion("1.0")]  
public class A {  
   public void M1() {}  
}  
  
public class C1 {}  
```  
  
## <a name="example"></a>예제  
 이 코드에서는 CS1701_d 어셈블리의 최신 버전을 만듭니다. 출력 파일에 같은 이름이 필요하므로 이전 버전과 다른 디렉터리에 컴파일됨에 유의하세요.  
  
```csharp  
// CS1701_b.cs  
// compile with: /target:library /out:c:\\cs1701_d.dll /keyfile:mykey.snk  
using System.Reflection;  
[assembly:AssemblyVersion("2.0")]  
public class A {  
   public void M2() {}  
   public void M1() {}  
}  
  
public class C2 {}  
public class C1 {}  
```  
  
## <a name="example"></a>예제  
 이 샘플은 A1 및 A2 외부 별칭을 설정합니다.  
  
```csharp  
// CS1701_c.cs  
// compile with: /target:library /reference:A2=c:\\cs1701_d.dll /reference:A1=cs1701_d.dll  
  
extern alias A1;  
extern alias A2;  
// using System;  
using a1 = A1::A;  
using a2 = A2::A;  
  
public class Ref {  
   public static a1 A1() { return new a1(); }  
   public static a2 A2() { return new a2(); }  
  
   public static A1::C1 M1() { return new A1::C1(); }  
   public static A2::C2 M2() { return new A2::C2(); }  
}  
```  
  
## <a name="example"></a>예제  
 이 샘플에서는 A의 두 개의 다른 별칭을 사용하여 메서드를 호출합니다. 다음 샘플에서는 C1701 오류가 발생합니다.  
  
```csharp  
// CS1701_d.cs  
// compile with: /reference:c:\\CS1701_d.dll /reference:CS1701_c.dll  
// CS1701 expected  
class Tester {  
   public static void Main() {  
      Ref.A1().M1();  
      Ref.A2().M2();  
   }  
}  
```
