<span data-ttu-id="9ce58-103">‘struct1’ 형식의 ‘struct2 field’ 구조체 멤버는 구조체 레이아웃에서 순환됩니다.</span><span class="sxs-lookup"><span data-stu-id="9ce58-103">Struct member 'struct2 field' of type 'struct1' causes a cycle in the struct layout</span></span>  
  
 <span data-ttu-id="9ce58-104">두 구조체의 정의에 재귀 참조가 포함되어 있습니다.</span><span class="sxs-lookup"><span data-stu-id="9ce58-104">The definitions of two structs include recursive references.</span></span> <span data-ttu-id="9ce58-105">각 구조체가 다른 구조체에서 자신을 정의하지 않도록 [struct](../../../csharp/language-reference/keywords/struct.md) 정의를 변경합니다.</span><span class="sxs-lookup"><span data-stu-id="9ce58-105">Change the [struct](../../../csharp/language-reference/keywords/struct.md) definitions such that each does not define itself on the other.</span></span> <span data-ttu-id="9ce58-106">구조체는 값 형식이므로 이 제한은 구조체에만 적용됩니다.</span><span class="sxs-lookup"><span data-stu-id="9ce58-106">This limitation applies only to structs, since structs are value types.</span></span> <span data-ttu-id="9ce58-107">재귀 참조를 사용하는 경우 형식을 클래스로 선언합니다.</span><span class="sxs-lookup"><span data-stu-id="9ce58-107">If using recursive references, declare your types as classes.</span></span>  
  
 <span data-ttu-id="9ce58-108">다음 샘플에서는 CS0523을 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="9ce58-108">The following sample generates CS0523:</span></span>  
  
```csharp  
// CS0523.cs  
// compile with: /target:library  
struct RecursiveLayoutStruct1  
{  
   public RecursiveLayoutStruct2 field;  
}  
  
struct RecursiveLayoutStruct2  
{  
   public RecursiveLayoutStruct1 field;   // CS0523  
}  
```
