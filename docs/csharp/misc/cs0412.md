# <a name="compiler-error-cs0412"></a><span data-ttu-id="f792c-102">컴파일러 오류 CS0412</span><span class="sxs-lookup"><span data-stu-id="f792c-102">Compiler Error CS0412</span></span>
<span data-ttu-id="f792c-103">'generic': 매개 변수 또는 지역 변수에는 메서드 형식 매개 변수와 같은 이름을 사용할 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="f792c-103">'generic': a parameter or local variable cannot have the same name as a method type parameter</span></span>  
  
 <span data-ttu-id="f792c-104">제네릭 메서드의 형식 매개 변수와 메서드 또는 메서드의 매개 변수 중 하나의 지역 변수 간에 이름 충돌이 발생합니다.</span><span class="sxs-lookup"><span data-stu-id="f792c-104">There is a name conflict between the type parameter of a generic method and a local variable in the method or one of the method's parameters.</span></span> <span data-ttu-id="f792c-105">이 오류를 방지하려면 모든 충돌하는 매개 변수 또는 지역 변수의 이름을 바꿉니다.</span><span class="sxs-lookup"><span data-stu-id="f792c-105">To avoid this error, rename any conflicting parameters or local variables.</span></span>  
  
## <a name="example"></a><span data-ttu-id="f792c-106">예</span><span class="sxs-lookup"><span data-stu-id="f792c-106">Example</span></span>  
 <span data-ttu-id="f792c-107">다음 샘플에서는 CS0412를 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="f792c-107">The following sample generates CS0412:</span></span>  
  
```csharp  
// CS0412.cs  
using System;  
  
class C  
{  
    // Parameter name is the same as method type parameter name  
    public void G<T>(int T)  // CS0412  
    {  
    }  
    public void F<T>()  
    {  
        // Method local variable name is the same as method type  
        // parameter name  
        double T = 0.0;  // CS0412  
        Console.WriteLine(T);  
    }  
  
    public static void Main()  
    {  
    }  
}  
```
