## <a name="example"></a><span data-ttu-id="d55e4-108">예제</span><span class="sxs-lookup"><span data-stu-id="d55e4-108">Example</span></span>  
 <span data-ttu-id="d55e4-109">다음 샘플에서는 CS0131을 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="d55e4-109">The following sample generates CS0131.</span></span>  
  
```csharp  
// CS0131.cs  
public class MyClass  
{  
    public int i = 0;  
    public void MyMethod()  
    {  
        i++ = 1;   // CS0131  
        // try the following line instead  
        // i = 1;  
    }  
    public static void Main() { }  
}  
```  
  
## <a name="example"></a><span data-ttu-id="d55e4-110">예제</span><span class="sxs-lookup"><span data-stu-id="d55e4-110">Example</span></span>  
 <span data-ttu-id="d55e4-111">다음 예제와 같이 대입 연산자의 왼쪽에서 산술 연산을 수행하려는 경우에도 이 오류가 발생할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="d55e4-111">This error can also occur if you attempt to perform arithmetic operations on the left hand side of an assignment operator, as in the following example.</span></span>

```csharp  
// CS0131b.cs

{  
    public static int Main()  
    {  
        int a = 1, b = 2, c = 3;  
        if (a + b = c) // CS0131  
        // try this instead  
        // if (a + b == c)  
            return 0;  
        return 1;  
    }  
}  
```
