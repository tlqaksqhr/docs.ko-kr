<span data-ttu-id="eee93-105">C#에는 `foreach` 및 `using`과 같이 정의된 패턴을 사용하는 문이 여러 개 있습니다.</span><span class="sxs-lookup"><span data-stu-id="eee93-105">There are several statements in C# that rely on defined patterns, such as `foreach` and `using`.</span></span> <span data-ttu-id="eee93-106">예를 들어는 [ `foreach` 문을](../language-reference/keywords/foreach-in.md) "열거 가능" 패턴을 구현 하는 컬렉션 클래스에 의존 합니다.</span><span class="sxs-lookup"><span data-stu-id="eee93-106">For example, the [`foreach` statement](../language-reference/keywords/foreach-in.md) relies on the collection class implementing the "enumerable" pattern.</span></span>
  
<span data-ttu-id="eee93-107">컴파일러에서 모호성으로 인해 일치시킬 수 없는 경우 CS0278이 발생할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="eee93-107">CS0278 can occur if the compiler is unable to make the match due to ambiguities.</span></span> <span data-ttu-id="eee93-108">예를 들어 "열거 가능" 패턴을 사용하려면 `MoveNext`메서드가 필요하며 코드에 `MoveNext`라는 메서드가 2개 포함될 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="eee93-108">For example, the "enumerable" pattern requires that there be a method called `MoveNext`, and your code might contain two methods called `MoveNext`.</span></span> <span data-ttu-id="eee93-109">컴파일러는 사용할 인터페이스를 찾으려고 하지만 모호성의 원인을 확인하여 해결하는 것이 좋습니다.</span><span class="sxs-lookup"><span data-stu-id="eee93-109">The compiler will attempt to find an interface to use, but it is recommended that you determine and resolve the cause of the ambiguity.</span></span>  
  
## <a name="example"></a><span data-ttu-id="eee93-110">예제</span><span class="sxs-lookup"><span data-stu-id="eee93-110">Example</span></span>

   public static void TestForeach<W>(W w)
