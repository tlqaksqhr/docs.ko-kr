<span data-ttu-id="a6d34-103">'global::'은 별칭이 아니라 전역 네임스페이스를 항상 참조하므로 별칭 이름을 'global'로 정의하지 않는 것이 좋습니다.</span><span class="sxs-lookup"><span data-stu-id="a6d34-103">Defining an alias named 'global' is ill-advised since 'global::' always references the global namespace and not an alias</span></span>  
  
 <span data-ttu-id="a6d34-104">이 경고는 전역이라는 별칭을 정의할 때 발생합니다.</span><span class="sxs-lookup"><span data-stu-id="a6d34-104">This warning is issued when you define an alias named global.</span></span>  
  
## <a name="example"></a><span data-ttu-id="a6d34-105">예</span><span class="sxs-lookup"><span data-stu-id="a6d34-105">Example</span></span>  
 <span data-ttu-id="a6d34-106">다음 예제에서는 CS0440을 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="a6d34-106">The following example generates CS0440:</span></span>  
  
```csharp  
// CS0440.cs  
// Compile with: /W:1  
  
using global = MyClass;   // CS0440  
class MyClass  
{  
    static void Main()  
    {  
        // Note how global refers to the global namespace  
        // even though it is redefined above.  
        global::System.Console.WriteLine();  
    }  
}  
```
