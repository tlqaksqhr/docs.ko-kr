[CLS 규격 코드 작성](http://msdn.microsoft.com/library/4c705105-69a2-4e5e-b24e-0633bc32c7f3)합니다.</span><span class="sxs-lookup"><span data-stu-id="36f7a-106">For more information on CLS Compliance, see [Writing CLS-Compliant Code](http://msdn.microsoft.com/library/4c705105-69a2-4e5e-b24e-0633bc32c7f3).</span></span>  
  
 <span data-ttu-id="36f7a-107">이 경고가 발생하지 않게 하려면 특성을 제거하거나 어셈블리 수준 특성을 추가합니다.</span><span class="sxs-lookup"><span data-stu-id="36f7a-107">To get rid of this warning, remove the attribute or add the assembly level attribute.</span></span>  
  
## <a name="example"></a><span data-ttu-id="36f7a-108">예제</span><span class="sxs-lookup"><span data-stu-id="36f7a-108">Example</span></span>  
 <span data-ttu-id="36f7a-109">다음 예제는 CS3021을 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="36f7a-109">The following example generates CS3021:</span></span>  
  
```csharp  
// CS3021.cs  
using System;  
// Uncomment the following line to declare the assembly CLS Compliant,  
// and avoid the warning without removing the attribute on the class.  
//[assembly: CLSCompliant(true)]  
  
// Remove the next line to avoid the warning.  
[CLSCompliant(false)]               // CS3021  
public class C  
{  
    public static void Main()  
    {  
    }  
}  
```
