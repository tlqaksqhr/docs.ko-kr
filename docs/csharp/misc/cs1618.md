# <a name="compiler-error-cs1618"></a><span data-ttu-id="6db14-102">컴파일러 오류 CS1618</span><span class="sxs-lookup"><span data-stu-id="6db14-102">Compiler Error CS1618</span></span>
<span data-ttu-id="6db14-103">Conditional 특성이 있으므로 'method'를 사용하여 대리자를 만들 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="6db14-103">Cannot create delegate with 'method' because it has a Conditional attribute</span></span>  
  
 <span data-ttu-id="6db14-104">메서드가 일부 빌드에서 없을 수도 있기 때문에 조건부 메서드로 대리자를 만들 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="6db14-104">You cannot create a delegate with a conditional method because the method might not exist in some builds.</span></span>  
  
 <span data-ttu-id="6db14-105">다음 샘플에서는 CS1618을 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="6db14-105">The following sample generates CS1618:</span></span>  
  
```csharp  
// CS1618.cs  
using System;  
using System.Diagnostics;  
  
delegate void del();  
  
class MakeAnError {  
   public static void Main() {  
      del d = new del(ConditionalMethod);   // CS1618  
      // Invalid because on builds where DEBUG is not set,   
      // there will be no "ConditionalMethod".  
   }  
   // To fix the error, remove the next line:  
   [Conditional("DEBUG")]  
   public static void ConditionalMethod()   
   {  
      Console.WriteLine("Do something only in debug");  
   }  
}  
```
