# <a name="compiler-error-cs1525"></a><span data-ttu-id="99a83-102">컴파일러 오류 CS1525</span><span class="sxs-lookup"><span data-stu-id="99a83-102">Compiler Error CS1525</span></span>
<span data-ttu-id="99a83-103">식의 'character' 항이 잘못되었습니다.</span><span class="sxs-lookup"><span data-stu-id="99a83-103">Invalid expression term 'character'</span></span>  
  
 <span data-ttu-id="99a83-104">컴파일러가 식에서 잘못된 문자를 발견했습니다.</span><span class="sxs-lookup"><span data-stu-id="99a83-104">The compiler detected an invalid character in an expression.</span></span>  
  
 <span data-ttu-id="99a83-105">다음 샘플에서는 CS1525를 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="99a83-105">The following sample generates CS1525:</span></span>  
  
```  
// CS1525.cs  
class x  
{  
   public static void Main()  
   {  
      int i = 0;  
      i = i +   // OK - identifier  
      'c' +     // OK - character  
      (5) +     // OK - parenthesis  
      [ +       // CS1525, operator not a valid expression element  
      throw +   // CS1525, keyword not allowed in expression  
      void;     // CS1525, void not allowed in expression  
   }  
}  
```  
  
 <span data-ttu-id="99a83-106">다음 샘플과 같이 빈 레이블이 CS1525를 생성할 수도 있습니다.</span><span class="sxs-lookup"><span data-stu-id="99a83-106">An empty label can also generate CS1525, as in the following sample:</span></span>  
  
```  
// CS1525b.cs  
using System;  
public class MyClass  
{  
   public static void Main()  
   {  
      goto FoundIt;  
      FoundIt:      // CS1525  
      // Uncomment the following line to resolve:  
      // System.Console.Write("Hello");  
   }  
}  
```
