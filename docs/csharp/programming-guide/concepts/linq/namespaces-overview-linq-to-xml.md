## <a name="xml-names"></a><span data-ttu-id="5bdf4-104">XML 이름</span><span class="sxs-lookup"><span data-stu-id="5bdf4-104">XML Names</span></span>  
 <span data-ttu-id="5bdf4-105">XML 이름으로 인해 XML 프로그래밍이 복잡해지는 경우가 많습니다.</span><span class="sxs-lookup"><span data-stu-id="5bdf4-105">XML names are often a source of complexity in XML programming.</span></span> <span data-ttu-id="5bdf4-106">XML 이름은 XML 네임스페이스(또는 XML 네임스페이스 URI라고 함)와 로컬 이름으로 구성되어 있습니다.</span><span class="sxs-lookup"><span data-stu-id="5bdf4-106">An XML name consists of an XML namespace (also called an XML namespace URI) and a local name.</span></span> <span data-ttu-id="5bdf4-107">XML 네임스페이스는 [!INCLUDE[dnprdnshort](~/includes/dnprdnshort-md.md)] 기반 프로그램의 네임스페이스와 유사합니다.</span><span class="sxs-lookup"><span data-stu-id="5bdf4-107">An XML namespace is similar to a namespace in a [!INCLUDE[dnprdnshort](~/includes/dnprdnshort-md.md)]-based program.</span></span> <span data-ttu-id="5bdf4-108">XML 네임스페이스를 통해 요소와 특성의 이름을 고유하게 정규화할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="5bdf4-108">It enables you to uniquely qualify the names of elements and attributes.</span></span> <span data-ttu-id="5bdf4-109">이에 따라 XML 문서의 다양한 부분 간에 이름 충돌이 방지됩니다.</span><span class="sxs-lookup"><span data-stu-id="5bdf4-109">This helps avoid name conflicts between various parts of an XML document.</span></span> <span data-ttu-id="5bdf4-110">XML 네임스페이스를 선언한 경우 해당 네임스페이스에서만 고유해야 하는 로컬 이름을 선택할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="5bdf4-110">When you have declared an XML namespace, you can select a local name that only has to be unique within that namespace.</span></span>  
  
 <span data-ttu-id="5bdf4-111">XML 이름의 또 다른 측면은 XML *네임스페이스 접두사*입니다.</span><span class="sxs-lookup"><span data-stu-id="5bdf4-111">Another aspect of XML names is XML *namespace prefixes*.</span></span> <span data-ttu-id="5bdf4-112">XML 접두사는 XML 이름이 복잡해지는 주된 원인입니다.</span><span class="sxs-lookup"><span data-stu-id="5bdf4-112">XML prefixes cause most of the complexity of XML names.</span></span> <span data-ttu-id="5bdf4-113">이러한 접두사를 통해 XML 네임스페이스의 바로 가기를 만들 수 있으며 이에 따라 XML 문서가 간결하고 이해하기 쉬워집니다.</span><span class="sxs-lookup"><span data-stu-id="5bdf4-113">These prefixes enable you to create a shortcut for an XML namespace, which makes the XML document more concise and understandable.</span></span> <span data-ttu-id="5bdf4-114">그러나 XML 접두사는 의미를 갖기 위해 컨텍스트에 의존하므로 복잡성이 가중됩니다.</span><span class="sxs-lookup"><span data-stu-id="5bdf4-114">However, XML prefixes depend on their context to have meaning, which adds complexity.</span></span> <span data-ttu-id="5bdf4-115">예를 들어, XML 접두사 `aw`를 XML 트리의 한 부분에 있는 한 XML 네임스페이스와 연결하고 XML 트리의 다른 부분에 있는 다른 XML 네임스페이스와 연결할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="5bdf4-115">For example, the XML prefix `aw` could be associated with one XML namespace in one part of an XML tree, and with a different XML namespace in a different part of the XML tree.</span></span>  
  
 <span data-ttu-id="5bdf4-116">C#과 함께 [!INCLUDE[sqltecxlinq](~/includes/sqltecxlinq-md.md)]을 사용하는 경우의 이점 중 하나는 XML 접두사를 사용할 필요가 없다는 것입니다.</span><span class="sxs-lookup"><span data-stu-id="5bdf4-116">One of the advantages of using [!INCLUDE[sqltecxlinq](~/includes/sqltecxlinq-md.md)] with C# is that you do not have to use XML prefixes.</span></span> <span data-ttu-id="5bdf4-117">
          [!INCLUDE[sqltecxlinq](~/includes/sqltecxlinq-md.md)]에서 XML 문서를 로드하고 구문을 분석할 때 각 XML 접두사는 해당하는 XML 네임스페이스로 확인됩니다.</span><span class="sxs-lookup"><span data-stu-id="5bdf4-117">When [!INCLUDE[sqltecxlinq](~/includes/sqltecxlinq-md.md)] loads or parses an XML document, each XML prefix is resolved to its corresponding XML namespace.</span></span> <span data-ttu-id="5bdf4-118">그 후 네임스페이스를 사용하는 문서로 작업할 때는 네임스페이스 접두사를 통해서가 아니라 거의 항상 네임스페이스 URI를 통해 네임스페이스에 액세스합니다.</span><span class="sxs-lookup"><span data-stu-id="5bdf4-118">After that, when you work with a document that uses namespaces, you almost always access the namespaces through the namespace URI, and not through the namespace prefix.</span></span> <span data-ttu-id="5bdf4-119">개발자는 [!INCLUDE[sqltecxlinq](~/includes/sqltecxlinq-md.md)]에서 XML 이름으로 작업할 때 항상 정규화된 XML 이름(즉, XML 네임스페이스 및 로컬 이름)으로 작업합니다.</span><span class="sxs-lookup"><span data-stu-id="5bdf4-119">When developers work with XML names in [!INCLUDE[sqltecxlinq](~/includes/sqltecxlinq-md.md)] they always work with a fully-qualified XML name (that is, an XML namespace and a local name).</span></span> <span data-ttu-id="5bdf4-120">그러나 필요한 경우 [!INCLUDE[sqltecxlinq](~/includes/sqltecxlinq-md.md)]에서 네임스페이스 접두사로 작업하고 네임스페이스 접두사를 제어할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="5bdf4-120">However, when necessary, [!INCLUDE[sqltecxlinq](~/includes/sqltecxlinq-md.md)] allows you to work with and control namespace prefixes.</span></span>  
  
 <span data-ttu-id="5bdf4-121">[!INCLUDE[sqltecxlinq](~/includes/sqltecxlinq-md.md)]에서 XML 이름을 나타내는 클래스는 <xref:System.Xml.Linq.XName>입니다.</span><span class="sxs-lookup"><span data-stu-id="5bdf4-121">In [!INCLUDE[sqltecxlinq](~/includes/sqltecxlinq-md.md)], the class that represents XML names is <xref:System.Xml.Linq.XName>.</span></span> <span data-ttu-id="5bdf4-122">XML 이름은 [!INCLUDE[sqltecxlinq](~/includes/sqltecxlinq-md.md)] API 전반에서 자주 나타나며, XML 이름이 필요할 때마다 <xref:System.Xml.Linq.XName> 매개 변수를 찾을 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="5bdf4-122">XML names appear frequently throughout the [!INCLUDE[sqltecxlinq](~/includes/sqltecxlinq-md.md)] API, and wherever an XML name is required, you will find an <xref:System.Xml.Linq.XName> parameter.</span></span> <span data-ttu-id="5bdf4-123">그러나 <xref:System.Xml.Linq.XName>으로 직접 작업하는 경우는 드뭅니다.</span><span class="sxs-lookup"><span data-stu-id="5bdf4-123">However, you rarely work directly with an <xref:System.Xml.Linq.XName>.</span></span> <span data-ttu-id="5bdf4-124"><xref:System.Xml.Linq.XName>에는 문자열에서의 암시적 변환이 포함되어 있습니다.</span><span class="sxs-lookup"><span data-stu-id="5bdf4-124"><xref:System.Xml.Linq.XName> contains an implicit conversion from string.</span></span>  
  
 <span data-ttu-id="5bdf4-125">자세한 내용은 <xref:System.Xml.Linq.XNamespace> 및 <xref:System.Xml.Linq.XName>를 참조하세요.</span><span class="sxs-lookup"><span data-stu-id="5bdf4-125">For more information, see <xref:System.Xml.Linq.XNamespace> and <xref:System.Xml.Linq.XName>.</span></span>
