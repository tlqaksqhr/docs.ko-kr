# <a name="what-is-docker"></a><span data-ttu-id="22bcf-103">Docker란?</span><span class="sxs-lookup"><span data-stu-id="22bcf-103">What is Docker?</span></span>

<span data-ttu-id="22bcf-104">[Docker](https://www.docker.com/)는 클라우드 또는 온-프레미스로 실행될 수 있는 이식 가능하고 문제를 스스로 해결할 수 있는 컨테이너로서 응용 프로그램 배포를 자동화하기 위한 [오픈 소스 프로젝트](https://github.com/docker/docker)입니다.</span><span class="sxs-lookup"><span data-stu-id="22bcf-104">[Docker](https://www.docker.com/) is an [open-source project](https://github.com/docker/docker) for automating the deployment of applications as portable, self-sufficient containers that can run on the cloud or on-premises.</span></span> <span data-ttu-id="22bcf-105">Docker는 이 기술을 장려하고 발전시키는 [회사](https://www.docker.com/)이기도 합니다.</span><span class="sxs-lookup"><span data-stu-id="22bcf-105">Docker is also a [company](https://www.docker.com/) that promotes and evolves this technology.</span></span> <span data-ttu-id="22bcf-106">Docker는 Microsoft를 비롯한 클라우드, Linux 및 Windows 공급 업체와 공동으로 작업합니다.</span><span class="sxs-lookup"><span data-stu-id="22bcf-106">Docker works in collaboration with cloud, Linux, and Windows vendors, including Microsoft.</span></span>

(macOS에서는 코드를 편집하거나 Docker CLI를 실행할 수 있지만 이 문서를 작성하는 시점에서는 컨테이너가 macOS에서 직접 실행되지 않습니다.) Windows에서 Linux 또는 Windows 컨테이너에 대한 이미지를 만들 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="22bcf-114">On Linux or macOS, you use a Docker host that is Linux based and can create images only for Linux containers.(On macOS you can edit code or run the Docker CLI, but as of the time of this writing, containers do not run directly on macOS.) <span data-ttu-id="22bcf-118">[Windows 컨테이너](https://msdn.microsoft.com/en-us/virtualization/windowscontainers/about/about_overview)는 두 가지 유형의 런타임에서 작동합니다.</span><span class="sxs-lookup"><span data-stu-id="22bcf-118">[Windows Containers](https://msdn.microsoft.com/en-us/virtualization/windowscontainers/about/about_overview) work with two types of runtimes:</span></span>

-   <span data-ttu-id="22bcf-119">Windows Server 컨테이너는 프로세스 및 네임스페이스 격리 기술을 통해 응용 프로그램 격리를 제공합니다.</span><span class="sxs-lookup"><span data-stu-id="22bcf-119">Windows Server Containers provide application isolation through process and namespace isolation technology.</span></span> <span data-ttu-id="22bcf-120">Windows Server 컨테이너는 컨테이너 호스트와 호스트에서 실행 중인 모든 컨테이너와 커널을 공유합니다.</span><span class="sxs-lookup"><span data-stu-id="22bcf-120">A Windows Server Container shares a kernel with the container host and with all containers running on the host.</span></span>

-   <span data-ttu-id="22bcf-121">Hyper-V 컨테이너는 고도로 최적화된 가상 머신에서 각 컨테이너를 실행하여 Windows Server 컨테이너가 제공하는 격리를 확장합니다.</span><span class="sxs-lookup"><span data-stu-id="22bcf-121">Hyper-V Containers expand on the isolation provided by Windows Server Containers by running each container in a highly optimized virtual machine.</span></span> <span data-ttu-id="22bcf-122">이 구성에서 컨테이너 호스트의 커널은 Hyper-V 컨테이너와 공유되지 않으므로 격리 기능이 향상됩니다.</span><span class="sxs-lookup"><span data-stu-id="22bcf-122">In this configuration, the kernel of the container host is not shared with the Hyper-V Containers, providing better isolation.</span></span> <span data-ttu-id="22bcf-123">Hyper-V 컨테이너를 사용하면 신뢰할 수 없고 *유해한 다중 테넌트* 응용 프로그램을 동일한 호스트에서 실행할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="22bcf-123">Hyper-V Containers allow untrusted and *hostile multi-tenant* applications to run on the same host.</span></span> <span data-ttu-id="22bcf-124">Hyper-V 컨테이너의 시작 시간과 밀도는 Windows Server 컨테이너보다 약간 비효율적입니다.</span><span class="sxs-lookup"><span data-stu-id="22bcf-124">Hyper-V Containers have a bit less efficiency in startup times and density than Windows Server Containers.</span></span>

<span data-ttu-id="22bcf-125">이러한 컨테이너의 이미지는 동일한 방식으로 생성되어 작동합니다.</span><span class="sxs-lookup"><span data-stu-id="22bcf-125">The images for these containers are created and function the same way.</span></span> <span data-ttu-id="22bcf-126">컨테이너가 생성되는 방식만 다릅니다.</span><span class="sxs-lookup"><span data-stu-id="22bcf-126">They differ in how the container is created.</span></span> <span data-ttu-id="22bcf-127">자세한 내용은 [Hyper-V 컨테이너](https://msdn.microsoft.com/en-us/virtualization/windowscontainers/about/about_overview)를 참조하세요.</span><span class="sxs-lookup"><span data-stu-id="22bcf-127">For details, see [Hyper-V Containers](https://msdn.microsoft.com/en-us/virtualization/windowscontainers/about/about_overview).</span></span>
