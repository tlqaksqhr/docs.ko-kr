# <a name="f-style-guide"></a>F # 스타일 가이드

다음 문서는 F # 코드 서식 지정 및 언어의 기능에 대 한 주제별 지침 및 사용 하는 방법에 대 한 지침을 설명 합니다.

이 지침을 기반으로 공식화 된에 프로그래머의 다양 한 그룹을 사용 하 여 코드 베이스에서 대규모의 F # 사용 합니다. 이 설명서는 일반적으로 F #를 성공적으로 사용 하면 하 고 프로그램에 대 한 요구 사항이 시간이 지남에 따라 변경 하는 경우 불만 최소화 합니다.

## <a name="five-principles-of-good-f-code"></a>적합 한 F # 코드의 다섯 가지 원칙

유지 해야 다음 원칙 염두에서 시간이 지남에 따라 변경 되는 시스템에서 특히 F # 코드를 작성 하 든 지 합니다. 모든 부분은 향후 문서에서 지침이 5 지점에서 기인합니다.

1. **적절 한 F # 코드는 간결 하 고 표현**

    F #에 더 적은 줄의 코드로 작업 express 및 일반 기능을 다시 사용할 수 있도록 하는 많은 기능이 있습니다. F # 핵심 라이브러리는 많은 유용한 형식 및 일반적인 데이터 컬렉션을 사용 하 여 작업에 대 한 함수에도 포함 합니다. 일반적으로 더 적은 줄 코드의 문제에 솔루션을 표현할 수 있으면 다른 개발자 (또는 이후 자신) 됩니다 감사의. 또한 좋습니다 FSharp.Core의 같은 라이브러리를 사용 하는 합니다 [방대한.NET 라이브러리](https://docs.microsoft.com/dotnet/api/) F #을 실행 하는 또는 타사 패키지에 [NuGet](https://www.nuget.org/) 중요 한 작업을 수행 해야 하는 경우.

2. **적합 한 F # 코드는 상호 운용이 가능**

    상호 운용성 등 서로 다른 언어로 코드를 사용 하는 여러 개의 폼을 걸릴 수 있습니다. 다른 호출자에 게 상호 운용 하는 코드의 경계가 까다로웠습니다. 중요 한 부분입니다. F #을 작성할 때 해야 항상 생각할 있습니다 어떻게 다른 코드에 대 한 코드를 작성 하는 C#과 같은 다른 언어에서 그렇게 하는 경우를 포함 하 여 호출 됩니다. 합니다 [F # 구성 요소 디자인 지침](component-design-guidelines.md) 자세히 상호 운용성에 설명 합니다.

3. **적절 한 F # 코드에서는 개체 프로그래밍을 하지 개체 방향**

    F #.net에서 개체를 사용한 프로그래밍에 대 한 전체 지원에 포함 [클래스](../language-reference/classes.md)를 [인터페이스](../language-reference/interfaces.md)합니다 [액세스 한정자](../language-reference/access-control.md), [추상 클래스](../language-reference/abstract-classes.md)등에입니다. 컨텍스트 인식 해야 하는 함수와 같은 더 복잡 한 함수 코드에 대 한 개체 컨텍스트 정보 함수 없는 방식으로 쉽게 캡슐화 됩니다. 같은 기능 [선택적 매개 변수](../language-reference/members/methods.md#optional-arguments) 하 고 신중 하 게 활용할 [오버 로드](../language-reference/members/methods.md#overloaded-methods) 쉽게 수행할 수 있습니다이 기능의 소비 호출자에 대 한 합니다.

4. **Mutation 노출 하지 않고 잘 수행 하는 좋은 F # 코드**

    아닙니다. 고성능 코드를 작성 하려면 변형을 사용 해야 합니다. 컴퓨터의 작동 원리, 결국 이며 이러한 코드는 이해 하기 어렵고 오류가 발생 하기 쉬운 경우가 많습니다. Mutation 호출자에 게 노출 하지 마십시오. 대신 [변이 기반 구현을 숨기는 기능 인터페이스를 구축](conventions.md#performance) 성능이 중요 한 경우.

5. **적절 한 F # 코드는 사항에 신경 쓰지**

    도구 작업 대규모 코드 베이스를 F # 언어 도구를 사용 하 여 보다 효율적으로 사용할 수 있도록 F # 코드를 작성할 수에 대 한 중요 한지 않습니다. 예로 있는지 확인 하면 지나치면 지점 무료 스타일이 프로그래밍, 디버거를 사용 하 여 중간 값을 검사할 수 있도록 합니다. 또 다른 예로 사용 하 여 [XML 문서 주석](../language-reference/xml-documentation.md) 편집기의 도구 설명 호출 사이트에서 이러한 의견을 표시할 수 있도록 구문을 설명 합니다. 항상 어떻게 코드, 탐색, 디버깅를 읽고 다른 프로그래머가 자신의 도구를 사용 하 여 조작 하는 방법에 대 한 생각 합니다.
