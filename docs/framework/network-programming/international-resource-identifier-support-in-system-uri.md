# <a name="international-resource-identifier-support-in-systemuri"></a><span data-ttu-id="2214a-102">System.Uri의 국가별 리소스 식별자 지원</span><span class="sxs-lookup"><span data-stu-id="2214a-102">International Resource Identifier Support in System.Uri</span></span>
<span data-ttu-id="2214a-103"><xref:System.Uri?displayProperty=nameWithType> 클래스는 IRI(International Resource Identifier) 및 IDN(Internationalized Domain Names)을 통해 확장되었습니다.</span><span class="sxs-lookup"><span data-stu-id="2214a-103">The <xref:System.Uri?displayProperty=nameWithType> class has been extended with International Resource Identifier (IRI) and Internationalized Domain Names (IDN) support.</span></span> <span data-ttu-id="2214a-104">이러한 향상된 기능은 .NET Framework 3.5, 3.0 SP1 및 2.0 SP1에서 사용할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="2214a-104">These enhancements are available in .NET Framework 3.5, 3.0 SP1, and 2.0 SP1.</span></span>  
  
## <a name="iri-and-idn-support"></a><span data-ttu-id="2214a-105">IRI 및 IDN 지원</span><span class="sxs-lookup"><span data-stu-id="2214a-105">IRI and IDN Support</span></span>  
 <span data-ttu-id="2214a-106">웹 주소는 대개 매우 제한적인 문자 집합으로 구성된 URI(Uniform Resource Identifier)를 사용하여 표현됩니다.</span><span class="sxs-lookup"><span data-stu-id="2214a-106">Web addresses are typically expressed using Uniform Resource Identifiers (URI) that consist of a very restricted set of characters:</span></span>

-   <span data-ttu-id="2214a-108">0~9의 숫자.</span><span class="sxs-lookup"><span data-stu-id="2214a-108">Digits from 0 to 9.</span></span>  
  
-   <span data-ttu-id="2214a-109">일부 기타 ASCII 기호.</span><span class="sxs-lookup"><span data-stu-id="2214a-109">A small number of other ASCII symbols.</span></span>  
  
 <span data-ttu-id="2214a-110">URI 사양은 IETF(Internet Engineering Task Force)에서 게시한 RFC 2396 및 RFC 3986에 설명되어 있습니다.</span><span class="sxs-lookup"><span data-stu-id="2214a-110">The specifications for URIs are documented in RFC 2396 and RFC 3986 published by the Internet Engineering Task Force (IETF).</span></span>  
  
 <span data-ttu-id="2214a-111">인터넷이 성장하면서 영어 이외의 언어로 리소스를 식별해야 할 필요성이 증가하고 있습니다.</span><span class="sxs-lookup"><span data-stu-id="2214a-111">With the growth of the Internet, there is a growing need to identify resources using languages other than English.</span></span> <span data-ttu-id="2214a-112">이 필요성을 가능하게 하고 ASCII 이외 문자(유니코드/ISO 10646 문자 집합의 문자)를 허용하는 식별자를 IRI(International Resource Identifier)라고 합니다.</span><span class="sxs-lookup"><span data-stu-id="2214a-112">Identifiers which facilitate this need and allow non-ASCII characters (characters in the Unicode/ISO 10646 character set) are known as International Resource Identifiers (IRIs).</span></span> <span data-ttu-id="2214a-113">IRI 사양은 IETF에서 게시한 RFC 3987에 설명되어 있습니다.</span><span class="sxs-lookup"><span data-stu-id="2214a-113">The specifications for IRIs are documented in RFC 3987 published by IETF.</span></span> <span data-ttu-id="2214a-114">IRI를 사용하면 URL에 유니코드 문자를 포함할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="2214a-114">Using IRIs allows a URL to contain Unicode characters.</span></span>  
  
 <span data-ttu-id="2214a-115">기존 <xref:System.Uri?displayProperty=nameWithType> 클래스는 RFC 3987을 기반으로 IRI 지원을 제공하도록 확장되었습니다.</span><span class="sxs-lookup"><span data-stu-id="2214a-115">The existing <xref:System.Uri?displayProperty=nameWithType> class has been extended to provide IRI support based on RFC 3987.</span></span> <span data-ttu-id="2214a-116">현재 사용자의 경우 IRI를 사용하도록 설정하지 않는 한 .NET Framework 2.0 동작에서 차이를 느끼지 못할 것입니다.</span><span class="sxs-lookup"><span data-stu-id="2214a-116">Current users will not see any change from the .NET Framework 2.0 behavior unless they specifically enable IRI.</span></span> <span data-ttu-id="2214a-117">이 덕분에 .NET Framework 이전 버전과의 응용 프로그램 호환성이 제공됩니다.</span><span class="sxs-lookup"><span data-stu-id="2214a-117">This ensures application compatibility with prior versions of the .NET Framework.</span></span>

 <span data-ttu-id="2214a-120">IDN을 사용하면 도메인 이름의 모든 유니코드 레이블이 해당 Punycode 문자로 변환됩니다.</span><span class="sxs-lookup"><span data-stu-id="2214a-120">Enabling IDN will convert all Unicode labels in a domain name to their Punycode equivalents.</span></span> <span data-ttu-id="2214a-121">Punycode 이름에는 ASCII 문자만 사용되며 항상 xn-- 접두사로 시작합니다.</span><span class="sxs-lookup"><span data-stu-id="2214a-121">Punycode names contain only ASCII characters and always start with the xn-- prefix.</span></span> <span data-ttu-id="2214a-122">대부분의 DNS 서버는 ASCII 문자만 지원하므로(RFC 3940 참조) 이렇게 해야 인터넷에서 기존 DNS 서버를 지원할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="2214a-122">The reason for this is to support existing DNS servers on the Internet, since most DNS servers only support ASCII characters (see RFC 3940).</span></span>  
  
 <span data-ttu-id="2214a-123">IRI 및 IDN을 사용하도록 설정하면 <xref:System.Uri.DnsSafeHost%2A?displayProperty=nameWithType> 속성 값에 영향을 줍니다.</span><span class="sxs-lookup"><span data-stu-id="2214a-123">Enabling IRI and IDN affects the value of the <xref:System.Uri.DnsSafeHost%2A?displayProperty=nameWithType> property.</span></span> <span data-ttu-id="2214a-124">IRI 및 IDN을 사용하면 <xref:System.Uri.Equals%2A?displayProperty=nameWithType>, <xref:System.Uri.OriginalString%2A?displayProperty=nameWithType>, <xref:System.Uri.GetComponents%2A?displayProperty=nameWithType> 및 <xref:System.Uri.IsWellFormedOriginalString%2A> 메서드의 동작을 변경할 수도 있습니다.</span><span class="sxs-lookup"><span data-stu-id="2214a-124">Enabling IRI and IDN can also change the behavior of the <xref:System.Uri.Equals%2A?displayProperty=nameWithType>, <xref:System.Uri.OriginalString%2A?displayProperty=nameWithType>, <xref:System.Uri.GetComponents%2A?displayProperty=nameWithType>, and <xref:System.Uri.IsWellFormedOriginalString%2A> methods.</span></span>

 <span data-ttu-id="2214a-129"><xref:System.GenericUriParserOptions.Idn?displayProperty=nameWithType> 형식은 파서가 호스트 이름의 IDN(Internationalized Domain Name) 구문 분석을 지원함을 나타냅니다.</span><span class="sxs-lookup"><span data-stu-id="2214a-129">The <xref:System.GenericUriParserOptions.Idn?displayProperty=nameWithType> type indicates the parser supports Internationalized Domain Name (IDN) parsing (IDN) of host names.</span></span> <span data-ttu-id="2214a-130">IDN이 실제로 사용되는지 여부는 IDN이 사용하도록 설정되었는지에 따라 다릅니다.</span><span class="sxs-lookup"><span data-stu-id="2214a-130">Whether IDN is actually used depends on if IDN is enabled.</span></span>
