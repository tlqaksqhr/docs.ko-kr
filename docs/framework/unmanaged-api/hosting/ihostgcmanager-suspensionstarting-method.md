# <a name="ihostgcmanagersuspensionstarting-method"></a>IHostGCManager::SuspensionStarting 메서드
공용 언어 런타임 (CLR) 가비지 수집을 수행 하기 위해 작업 실행을 중단 하는 호스트에 알립니다.  
  
## <a name="syntax"></a>구문  
  
```  
HRESULT SuspensionStarting ();  
```  
  
## <a name="return-value"></a>반환 값  
  
|HRESULT|설명|  
|-------------|-----------------|  
|S_OK|`SuspensionStarting` 성공적으로 반환 합니다.|  
|HOST_E_CLRNOTAVAILABLE|CLR은 프로세스에 로드 되지 않았습니다 또는 CLR 중인 상태를 관리 코드를 실행 하거나 호출을 처리할 수 없습니다.|  
|HOST_E_TIMEOUT|호출 시간이 초과 되었습니다.|  
|HOST_E_NOT_OWNER|호출자에 게 잠금을 소유 하지 않습니다.|  
|HOST_E_ABANDONED|차단 된 스레드 이벤트 취소 되었습니다 또는 파이버가 기다리던 합니다.|  
|E_FAIL|알 수 없는 치명적인 오류가 발생 했습니다. 메서드가 E_FAIL을 반환 하는 경우 CLR을 하는 프로세스 내에서 사용할 수 없습니다. 호스팅 방법에 대 한 후속 호출 HOST_E_CLRNOTAVAILABLE를 반환 합니다.|  
  
## <a name="remarks"></a>설명  
 CLR에서는 `SuspensionStarting` 가비지 수집이 발생 하는 호스트입니다.  
  
> [!IMPORTANT]
>  이 작업의 일정을 재조정 마십시오. 호스트는 작업을 다시 예약 해야 때 [ThreadIsBlockingForSuspension](../../../../docs/framework/unmanaged-api/hosting/ihostgcmanager-threadisblockingforsuspension-method.md) 호출 됩니다.
