# <a name="entity-data-model-inheritance"></a><span data-ttu-id="a41c1-102">엔터티 데이터 모델: 상속</span><span class="sxs-lookup"><span data-stu-id="a41c1-102">Entity Data Model: Inheritance</span></span>
<span data-ttu-id="a41c1-103">(EDM (엔터티 데이터 모델)에 대 한 상속 지원 [엔터티 형식](../../../../docs/framework/data/adonet/entity-type.md)합니다.</span><span class="sxs-lookup"><span data-stu-id="a41c1-103">The Entity Data Model (EDM) supports inheritance for [entity types](../../../../docs/framework/data/adonet/entity-type.md).</span></span> <span data-ttu-id="a41c1-104">EDM의 상속은 개체 지향 프로그래밍 언어의 클래스에 대한 상속과 유사합니다.</span><span class="sxs-lookup"><span data-stu-id="a41c1-104">Inheritance in the EDM is similar to inheritance for classes in object-oriented programming languages.</span></span> <span data-ttu-id="a41c1-105">개체 지향 언어의 클래스와 함께 개념적 모델에서 정의한 엔터티 형식 처럼 (한 *파생 형식*) 다른 엔터티 형식에서 상속 되는 (의 *기본 형식*).</span><span class="sxs-lookup"><span data-stu-id="a41c1-105">Like with classes in object-oriented languages, in a conceptual model you can define an entity type (a *derived type*) that inherits from another entity type (the *base type*).</span></span> <span data-ttu-id="a41c1-106">그러나 개체 지향 프로그래밍의 클래스와 달리 개념적 모델에서 파생 된 형식 항상 모든 상속 된 [속성](../../../../docs/framework/data/adonet/property.md) 및 [탐색 속성](../../../../docs/framework/data/adonet/navigation-property.md) 기본 형식입니다.</span><span class="sxs-lookup"><span data-stu-id="a41c1-106">However, unlike classes in object-oriented programming, in a conceptual model the derived type always inherits all the [properties](../../../../docs/framework/data/adonet/property.md) and [navigation properties](../../../../docs/framework/data/adonet/navigation-property.md) of the base type.</span></span> <span data-ttu-id="a41c1-107">파생 형식에서 상속된 속성을 재정의할 수는 없습니다.</span><span class="sxs-lookup"><span data-stu-id="a41c1-107">You cannot override inherited properties in a derived type.</span></span>  
  
 <span data-ttu-id="a41c1-108">개념적 모델에서는 파생 형식이 다른 파생 형식으로부터 상속받는 상속 계층 구조를 작성할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="a41c1-108">In a conceptual model you can build inheritance hierarchies in which a derived type inherits from another derived type.</span></span> <span data-ttu-id="a41c1-109">계층 (파생 형식이 아닌 계층의 한 유형)의 최상위에 있는 형식의 라고는 *루트 형식*합니다.</span><span class="sxs-lookup"><span data-stu-id="a41c1-109">The type at the top of the hierarchy (the one type in the hierarchy that is not a derived type) is called the *root type*.</span></span> <span data-ttu-id="a41c1-110">상속 계층 구조의 [엔터티 키](../../../../docs/framework/data/adonet/entity-key.md) 루트 형식에서 정의 되어야 합니다.</span><span class="sxs-lookup"><span data-stu-id="a41c1-110">In an inheritance hierarchy, the [entity key](../../../../docs/framework/data/adonet/entity-key.md) must be defined on the root type.</span></span>  
  
 <span data-ttu-id="a41c1-111">파생 형식이 둘 이상의 형식으로부터 상속받는 상속 계층 구조는 작성할 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="a41c1-111">You cannot build inheritance hierarchies in which a derived type inherits from more than one type.</span></span> <span data-ttu-id="a41c1-112">예를 들어, `Book` 엔터티 형식이 포함된 개념적 모델에서는 각각 `FictionBook`으로부터 상속되는 파생 형식 `NonFictionBook`과 `Book`을 정의할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="a41c1-112">For example, in a conceptual model with a `Book` entity type, you could define derived types `FictionBook` and `NonFictionBook` that each inherit from `Book`.</span></span> <span data-ttu-id="a41c1-113">그러나 `FictionBook` 및 `NonFictionBook` 형식에서 모두 상속되는 형식을 정의할 수는 없습니다.</span><span class="sxs-lookup"><span data-stu-id="a41c1-113">However, you could not then define a type that inherits from both the `FictionBook` and `NonFictionBook` types.</span></span>
