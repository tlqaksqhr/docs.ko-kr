 <span data-ttu-id="31ff6-109">삽입 프로세스는 텍스트 문자열을 중간에 종료하고 새 명령을 추가하는 방식으로 작동합니다.</span><span class="sxs-lookup"><span data-stu-id="31ff6-109">The injection process works by prematurely terminating a text string and appending a new command.</span></span> <span data-ttu-id="31ff6-110">삽입된 명령에는 실행되기 전에 추가된 문자열이 있을 수 있으므로 공격자는 주석 표시("--")를 사용하여 삽입된 문자열을 종료합니다.</span><span class="sxs-lookup"><span data-stu-id="31ff6-110">Because the inserted command may have additional strings appended to it before it is executed, the malefactor terminates the injected string with a comment mark "--".</span></span> <span data-ttu-id="31ff6-111">이후 텍스트는 실행 시 무시됩니다.</span><span class="sxs-lookup"><span data-stu-id="31ff6-111">Subsequent text is ignored at execution time.</span></span> <span data-ttu-id="31ff6-112">세미콜론(;) 구분 기호를 사용하여 여러 명령을 삽입할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="31ff6-112">Multiple commands can be inserted using a semicolon (;) delimiter.</span></span>  
  
 <span data-ttu-id="31ff6-113">삽입된 SQL 코드가 구문상 올바른 경우 프로그래밍 방식으로는 훼손 여부를 찾아낼 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="31ff6-113">As long as injected SQL code is syntactically correct, tampering cannot be detected programmatically.</span></span> <span data-ttu-id="31ff6-114">따라서 모든 사용자 입력의 유효성을 검사하고 생성된 SQL 명령을 현재 사용 중인 서버에서 실행하는 코드를 신중하게 검토해야 합니다.</span><span class="sxs-lookup"><span data-stu-id="31ff6-114">Therefore, you must validate all user input and carefully review code that executes constructed SQL commands in the server that you are using.</span></span> <span data-ttu-id="31ff6-115">유효성을 검사하지 않은 사용자 입력은 연결하지 마세요.</span><span class="sxs-lookup"><span data-stu-id="31ff6-115">Never concatenate user input that is not validated.</span></span> <span data-ttu-id="31ff6-116">문자열 연결은 스크립트 삽입 공격의 기본적인 대상이 됩니다.</span><span class="sxs-lookup"><span data-stu-id="31ff6-116">String concatenation is the primary point of entry for script injection.</span></span>  
  
 <span data-ttu-id="31ff6-117">다음은 몇 가지 유용한 지침입니다.</span><span class="sxs-lookup"><span data-stu-id="31ff6-117">Here are some helpful guidelines:</span></span>  
  
-   <span data-ttu-id="31ff6-118">Transact-SQL 문을 사용자 입력에서 직접 빌드하지 않고 저장 프로시저를 사용하여 사용자 입력의 유효성을 검사합니다.</span><span class="sxs-lookup"><span data-stu-id="31ff6-118">Never build Transact-SQL statements directly from user input; use stored procedures to validate user input.</span></span>  
  
-   <span data-ttu-id="31ff6-119">유형, 길이, 형식 및 범위를 테스트하여 사용자 입력의 유효성을 검사합니다.</span><span class="sxs-lookup"><span data-stu-id="31ff6-119">Validate user input by testing type, length, format, and range.</span></span> <span data-ttu-id="31ff6-120">Transact-SQL QUOTENAME() 함수를 사용하여 시스템 이름을 이스케이프하거나 REPLACE() 함수를 사용하여 문자열의 모든 문자를 이스케이프합니다.</span><span class="sxs-lookup"><span data-stu-id="31ff6-120">Use the Transact-SQL QUOTENAME() function to escape system names or the REPLACE() function to escape any character in a string.</span></span>  
  
-   <span data-ttu-id="31ff6-121">응용 프로그램의 각 계층에서 여러 단계의 유효성 검사를 구현합니다.</span><span class="sxs-lookup"><span data-stu-id="31ff6-121">Implement multiple layers of validation in each tier of your application.</span></span>  
  
-   <span data-ttu-id="31ff6-122">입력의 크기와 데이터 형식을 테스트하고 적절한 제한을 적용합니다.</span><span class="sxs-lookup"><span data-stu-id="31ff6-122">Test the size and data type of input and enforce appropriate limits.</span></span> <span data-ttu-id="31ff6-123">이렇게 하면 의도적인 버퍼 오버런을 방지할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="31ff6-123">This can help prevent deliberate buffer overruns.</span></span>

-   <span data-ttu-id="31ff6-127">다중 계층 환경에서는 신뢰할 수 있는 영역으로 들어가는 모든 데이터의 유효성을 검사해야 합니다.</span><span class="sxs-lookup"><span data-stu-id="31ff6-127">In multi-tiered environments, all data should be validated before admission to the trusted zone.</span></span>

-   <span data-ttu-id="31ff6-129"><xref:System.Data.SqlClient.SqlParameter> 개체를 저장 프로시저 및 명령과 함께 사용하여 형식 검사와 길이 유효성 검사를 제공합니다.</span><span class="sxs-lookup"><span data-stu-id="31ff6-129">Use <xref:System.Data.SqlClient.SqlParameter> objects with stored procedures and commands to provide type checking and length validation.</span></span>  
  
-   <span data-ttu-id="31ff6-130">클라이언트 코드에 <xref:System.Text.RegularExpressions.Regex> 식을 사용하여 유효하지 않은 문자를 필터링합니다.</span><span class="sxs-lookup"><span data-stu-id="31ff6-130">Use <xref:System.Text.RegularExpressions.Regex> expressions in client code to filter invalid characters.</span></span>
