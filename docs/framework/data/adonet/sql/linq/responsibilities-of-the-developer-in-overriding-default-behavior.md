-   <span data-ttu-id="4f2ca-104">재정의 메서드에서는 <xref:System.Data.Linq.DataContext.SubmitChanges%2A> 또는 <xref:System.Data.Linq.Table%601.Attach%2A>를 호출해서는 안 됩니다.</span><span class="sxs-lookup"><span data-stu-id="4f2ca-104">The overriding method must not call <xref:System.Data.Linq.DataContext.SubmitChanges%2A> or <xref:System.Data.Linq.Table%601.Attach%2A>.</span></span> <span data-ttu-id="4f2ca-105">이러한 메서드를 호출하면 [!INCLUDE[vbtecdlinq](../../../../../../includes/vbtecdlinq-md.md)]에서 예외를 throw합니다.</span><span class="sxs-lookup"><span data-stu-id="4f2ca-105">[!INCLUDE[vbtecdlinq](../../../../../../includes/vbtecdlinq-md.md)] throws an exception if these methods are called in an override method.</span></span>  
  
-   <span data-ttu-id="4f2ca-106">재정의 메서드는 트랜잭션을 시작, 커밋 또는 중지하는 데 사용할 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="4f2ca-106">Override methods cannot be used to start, commit, or stop a transaction.</span></span> <span data-ttu-id="4f2ca-107"><xref:System.Data.Linq.DataContext.SubmitChanges%2A> 작업은 트랜잭션에서 수행됩니다.</span><span class="sxs-lookup"><span data-stu-id="4f2ca-107">The <xref:System.Data.Linq.DataContext.SubmitChanges%2A> operation is performed under a transaction.</span></span> <span data-ttu-id="4f2ca-108">내부 중첩 트랜잭션이 외부 트랜잭션을 방해할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="4f2ca-108">An inner nested transaction can interfere with the outer transaction.</span></span> <span data-ttu-id="4f2ca-109">로드 재정의 메서드는 <xref:System.Transactions.Transaction>에서 작업이 수행되지 않는 것을 확인한 후에만 트랜잭션을 시작할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="4f2ca-109">Load override methods can start a transaction only after they determine that the operation is not being performed in a <xref:System.Transactions.Transaction>.</span></span>

-   <span data-ttu-id="4f2ca-113">만들기(`Insert`) 및 `Update` 재정의 메서드에서는 작업이 완료되면 데이터베이스에서 생성된 열의 값을 해당 개체 멤버로 다시 보내야 합니다.</span><span class="sxs-lookup"><span data-stu-id="4f2ca-113">Create (`Insert`) and `Update` override methods are expected to flow back the values for database-generated columns to corresponding object members when the operation is successfully completed.</span></span>

-   <span data-ttu-id="4f2ca-117">올바른 동적 API를 호출하는 것은 사용자의 책임입니다.</span><span class="sxs-lookup"><span data-stu-id="4f2ca-117">It is the user's responsibility to invoke the correct dynamic API.</span></span> <span data-ttu-id="4f2ca-118">예를 들어 업데이트 재정의 메서드에서는 <xref:System.Data.Linq.DataContext.ExecuteDynamicUpdate%2A>만 호출할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="4f2ca-118">For example, in the update override method, only the <xref:System.Data.Linq.DataContext.ExecuteDynamicUpdate%2A> can be called.</span></span> [!INCLUDE[vbtecdlinq](../../../../../../includes/vbtecdlinq-md.md)]<span data-ttu-id="4f2ca-119">에서는 호출된 동적 메서드가 적용 가능한 작업과 일치하는지 여부를 감지하거나 확인하지 않습니다.</span><span class="sxs-lookup"><span data-stu-id="4f2ca-119"> does not detect or verify whether the invoked dynamic method matches the applicable operation.</span></span> <span data-ttu-id="4f2ca-120">업데이트할 개체에 대해 <xref:System.Data.Linq.DataContext.ExecuteDynamicDelete%2A>와 같은 적용 불가능한 메서드를 호출하면 결과가 정의되지 않습니다.</span><span class="sxs-lookup"><span data-stu-id="4f2ca-120">If an inapplicable method is called (for example, <xref:System.Data.Linq.DataContext.ExecuteDynamicDelete%2A> for an object to be updated), the results are undefined.</span></span>  
  
-   <span data-ttu-id="4f2ca-121">마지막으로 재정의 메서드는 지정된 작업을 수행해야 합니다.</span><span class="sxs-lookup"><span data-stu-id="4f2ca-121">Finally, the overriding method is expected to perform the stated operation.</span></span> <span data-ttu-id="4f2ca-122">즉시 로드, 지연된 로드 및 [!INCLUDE[vbtecdlinq](../../../../../../includes/vbtecdlinq-md.md)]와 같은 <xref:System.Data.Linq.DataContext.SubmitChanges%2A> 작업의 의미가 구현되려면 재정의 메서드에서 해당 서비스를 제공해야 합니다.</span><span class="sxs-lookup"><span data-stu-id="4f2ca-122">The semantics of [!INCLUDE[vbtecdlinq](../../../../../../includes/vbtecdlinq-md.md)] operations such as eager loading, deferred loading, and <xref:System.Data.Linq.DataContext.SubmitChanges%2A>) require the overrides to provide the stated service.</span></span> <span data-ttu-id="4f2ca-123">예를 들어 데이터베이스의 내용을 확인하지 않고 빈 컬렉션만 반환하는 로드 재정의는 데이터 불일치를 발생시킬 수도 있습니다.</span><span class="sxs-lookup"><span data-stu-id="4f2ca-123">For example, a load override that just returns an empty collection without checking the contents in the database will likely lead to inconsistent data.</span></span>
