# 방법: 코드 편집기를 사용하여 엔터티 클래스 사용자 지정
[!INCLUDE[vs_current_short](../../../../../../includes/vs-current-short-md.md)]를 사용하는 개발자는 [!INCLUDE[vs_ordesigner_long](../../../../../../includes/vs-ordesigner-long-md.md)]를 사용하여 엔터티 클래스를 만들거나 사용자 지정할 수 있습니다.  
  
 [!INCLUDE[vsprvs](../../../../../../includes/vsprvs-md.md)] 코드 편집기를 사용하여 사용자 고유의 매핑 코드를 작성하거나 이미 생성된 코드를 사용자 지정할 수도 있습니다.  자세한 내용은 [특성 기반 매핑](../../../../../../docs/framework/data/adonet/sql/linq/attribute-based-mapping.md)을 참조하세요.
